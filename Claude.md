# CLAUDE.md

このファイルは Claude Code への薄い案内書。実作業の詳細は領域別 CLAUDE.md とスキルに委譲し、ここでは全体の姿勢・作業前の確認・進め方だけを持つ。

## このリポジトリの位置づけ

個人向けのレシピ管理アプリ。バックエンドは Go(Echo + GORM + PostgreSQL)、フロントエンドは React(Vite)。

## 基本姿勢

- **既存コードと git 履歴をお手本にする。** 設計方針は文書で縛らず、同じ領域の既存実装に倣う。迷ったら周辺のコードを読んで、その書き方に合わせる。
- **必要になったら都度変える。** 規約は固定物ではない。現状に合わなくなったら、黙って逸脱せず CLAUDE.md の更新を提案する。
- 明文化しておきたい不変式・コマンドは領域別 CLAUDE.md に集約する(下記)。

## 作業前の確認

不可逆な操作・課金・外向きの公開・権限・要件の曖昧さに関わる判断は、実行する前に停止して確認する。迷うときも停止する。

## Git・PR ワークフロー

- **変更前にブランチを切る。** 着手時の最初の所作として `git status` を見て、クリーンなら最新 `main` からブランチを切ってから実装に入る。コミット直前ではなく変更前に切る。
- **マージはスカッシュマージ。**
- commit・push・PR・マージの自走範囲(検討のみ / コミットまで / マージ可能まで)は指示に従う。未指定なら確認する。

## 領域別の規約

アプリ別のコマンド・不変式・慣習は各領域の CLAUDE.md に委譲する。対象ファイルの所在に応じて Claude Code が自動で読み込む。

- バックエンド: [api/CLAUDE.md](api/CLAUDE.md)
- フロントエンド: [frontend/CLAUDE.md](frontend/CLAUDE.md)

## レビュー

- **push 前に `/code-review`** を 1 パス通す(正確性バグ・簡素化)。
- 設計・規約に効く差分(サービス／ロジック層・新規モジュール・構造や依存方向の変更)を含むなら、加えて **`convention-review` スキル**を回す(規約・既存慣習への適合を第三者視点で見る)。瑣末な差分(ドキュメント・整形のみ)では省く。

## ドキュメント

Markdown ドキュメント(CLAUDE.md・README・設計メモ)や Issue・PR の文章を書く・整えるときは、`readable-docs` スキルの文章作法に従う。

## 動作確認・ビジュアル確認

UI の変更(特にレイアウト・モバイル対応・見た目)をしたときは、できるだけ実際に動かして確認する。状況に応じて使い分ける。

- **`/run`** … アプリを起動してブラウザで開き、スクリーンショットを撮る。「動いてるところを見せて」系。
- **`/verify`** … 変更が意図どおり動くかをアプリを起動して操作・観察して確認する。「この修正が効いているか確かめて」系。
- **Playwright**(`frontend/` に同梱) … 任意のビューポート幅(例: 375px のスマホ、768px のタブレット)でページを開き `screenshot()` で撮る。モバイル対応の確認に最適。API はバックエンド不要のルートモックで差し替えられる([frontend/e2e/recipes.spec.ts](frontend/e2e/recipes.spec.ts) の `mockApi` を参照)。

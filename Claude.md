## ADR（設計判断の参照）

作業を始める前に必ず [adr/README.md](adr/README.md) を読むこと。これが ADR の索引で、各 ADR へのリンクと運用ルールが書かれている。

- README の「一覧」にある ADR は**すべて目を通し、その決定に従って**実装・レビューする。
- ADR はレイヤーごとに分かれている。バックエンド(`api/`)の作業時は [adr/api/](adr/api/) の ADR、フロントエンド(`frontend/`)の作業時は [adr/frontend/](adr/frontend/) の ADR を参照する。
- 新しい設計判断をしたときは、README の命名規則・フォーマットに従って該当ディレクトリへ ADR を追加する。
